Perintah Git

Perintah Git dan GitHub

Apa itu Git?

Git adalah sistem kontrol versi terdistribusi untuk melacak perubahan pada kode sumber. Git memungkinkan banyak pengembang bekerja pada proyek yang sama tanpa menghapus pekerjaan orang lain.

Konsep Dasar Git

1. Modified

Modified adalah kondisi di mana file dalam direktori kerja telah diubah, tetapi perubahan tersebut belum ditambahkan ke Staging Area. File yang telah dimodifikasi hanya terlihat di direktori kerja hingga Anda menjalankan perintah git add.

2. Staging

Staging adalah proses di mana perubahan yang telah dilakukan pada file ditambahkan ke Staging Area menggunakan perintah git add. Staging memastikan bahwa hanya perubahan yang telah dipilih yang akan disertakan dalam commit.

3. Committed

Committed adalah keadaan di mana perubahan yang telah ditambahkan ke Staging Area disimpan secara permanen di repositori Git menggunakan perintah git commit. Setiap commit akan memiliki riwayat yang dapat dilacak.

Perintah Git Dasar

1. git init

Perintah ini digunakan untuk menginisialisasi repositori Git baru di direktori saat ini. Tujuannya adalah agar project kita dapat menjalankan versioning control system.

git init

2. git status

Perintah ini menampilkan status dari direktori kerja dan staging area, termasuk perubahan yang akan dikomit.

git status

3. git add

Ketika kita membuat sebuah perubahan di dalam project file kita maka perubahan itu akan masuk ke modified terlebih dahulu. Kita perlu memindahkan perubahan tersebut ke staging area. Tujuannya agar perubahan sudah tersimpan di staging area. Kita bisa menjalankan perintah git add nama_file namafiles atau git add ., biasanya jika kita tidak ingin memindahkan satu-satu file ke staging area kita menggunakan git add .

git add nama_file atau
git add .

4. git commit

Perintah git commit digunakan untuk menyimpan semua perubahan yang telah ditambahkan ke dalam repositori lokal. Ini memastikan file yang telah dimodifikasi tetap aman, sehingga Anda tidak perlu khawatir kehilangan perubahan saat berpindah branch, melakukan pull, atau merge repositori.

Anda dapat menggunakan perintah git commit atau git commit -m "pesan commit". Apa perbedaannya? Dengan git commit, Anda dapat menambahkan deskripsi yang lebih rinci, sedangkan git commit -m "pesan commit" adalah cara cepat (shortcut) untuk menambahkan pesan singkat saat melakukan commit.

git commit -m "Pesan commit Anda"

5. git remote

git remote berfungsi untuk menghubungkan repositori lokal dengan repositori jarak jauh (remote repository). Dengan menggunakan git remote, Anda dapat mengelola hubungan antara repositori lokal dan platform seperti GitHub, GitLab, atau Bitbucket. Perintah ini memungkinkan Anda untuk melakukan sinkronisasi perubahan, seperti push untuk mengunggah perubahan dari lokal ke remote atau pull untuk mengambil pembaruan dari remote ke lokal.

Contoh perintah dasar:

  • git remote add origin [URL]: Menambahkan repositori remote dengan nama origin.
  • git remote -v: Melihat daftar repositori remote yang terhubung.
  • git remote remove [name]: Menghapus repositori remote tertentu.

6. git push

Perintah git push digunakan untuk mengunggah atau mengirim perubahan dari repositori lokal ke repositori remote (contohnya GitHub, GitLab, atau Bitbucket). Setelah Anda membuat perubahan pada kode di lokal dan melakukan commit, perintah ini akan menyinkronkan repositori remote dengan semua commit terbaru Anda.

Dalam contoh git push origin main:

  • origin: Nama default dari repositori remote.
  • main: Nama cabang (branch) di repositori remote yang menjadi tujuan untuk mengunggah commit.

Pastikan Anda memiliki hak akses ke repositori remote sebelum menjalankan perintah ini.

git push origin main

7. git branch

Perintah git branch digunakan untuk mengelola cabang (branch) di repositori Git. Cabang memungkinkan Anda bekerja pada fitur atau perubahan baru tanpa memengaruhi kode utama.

Beberapa penggunaan umum perintah ini:

  • git branch: Menampilkan daftar semua cabang di repositori, dengan tanda bintang (*) menunjukkan cabang aktif.
  • git branch nama_cabang: Membuat cabang baru dengan nama nama_cabang.
  • git branch -d nama_cabang: Menghapus cabang nama_cabang (hanya jika sudah digabungkan ke cabang lain).

Cabang sering digunakan bersamaan dengan perintah git checkout atau git switch untuk berpindah antar cabang.

git branch nama_fitur

8. git merge

Perintah git merge digunakan untuk menggabungkan perubahan yang ada di cabang lain ke cabang aktif saat ini. Ini memungkinkan Anda untuk mengintegrasikan fitur atau perbaikan yang telah dibuat di cabang lain ke dalam cabang utama atau cabang yang sedang Anda kerjakan.

Beberapa penggunaan umum perintah ini:

  • git merge nama_cabang: Menggabungkan cabang nama_cabang ke cabang aktif saat ini.
  • git merge --no-ff nama_cabang: Melakukan penggabungan dengan membuat commit merge meskipun penggabungan dapat dilakukan dengan fast-forward.
  • git merge --abort: Membatalkan proses merge jika terjadi konflik dan Anda ingin kembali ke keadaan semula sebelum merge.

Perintah git merge sering digunakan setelah bekerja di cabang fitur untuk menggabungkan perubahan kembali ke cabang utama, seperti master atau main.

git merge fitur_baru

9. git log

Perintah ini menampilkan riwayat commit dari repositori.

git log

10. git diff

Perintah ini menampilkan perbedaan antara perubahan yang ada di direktori kerja dan yang ada di staging area.

git diff

11. git pull

Perintah ini digunakan untuk mengambil dan menggabungkan perubahan dari repositori remote ke repositori lokal Anda.

git pull origin main

Panduan GitHub

GitHub adalah layanan hosting untuk proyek Git berbasis web yang memungkinkan kolaborasi antara pengembang. Di GitHub, Anda dapat membuat repositori untuk proyek Anda, berkolaborasi dengan orang lain, serta melakukan versi kontrol dengan cara yang mudah.

1. Membuat Repositori di GitHub

Untuk memulai proyek di GitHub, Anda harus membuat repositori terlebih dahulu.

Langkah-langkah:

  1. Login ke GitHub dan klik tombol "New" di halaman repositori.
  2. Isi nama repositori dan pilih opsi lainnya sesuai kebutuhan.
  3. Klik "Create repository".

2. Menghubungkan Repositori Lokal dengan GitHub

Setelah membuat repositori di GitHub, Anda perlu menghubungkan repositori lokal Anda dengan repositori di GitHub menggunakan perintah git remote.

Langkah-langkah:

  1. Salin URL repositori GitHub.
  2. Gunakan perintah berikut untuk menghubungkan repositori lokal ke repositori GitHub:
    3. git remote add origin URL_GITHUB

3. Push dan Pull dari GitHub

Setelah repositori terhubung, Anda bisa menggunakan perintah git push untuk mengirim perubahan lokal ke GitHub, dan git pull untuk menarik perubahan terbaru dari GitHub.

Contoh perintah:

git push -u origin main

Perintah di atas digunakan untuk mengirim commit lokal ke repositori GitHub di cabang main.
